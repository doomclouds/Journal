# Journal 数据安全声明

本声明说明 Journal 当前版本对本地数据材料、缓存索引、导入导出和卸载保留的处理原则。

## 数据材料

Journal 将以下文件视为长期数据材料：

- Markdown 日记条目：正式日记内容。
- raw-input jsonl 文件：用户原始输入。
- version snapshot 文件：正式条目被覆盖前的版本快照。

上述文件用于记录、恢复和审计个人日记内容，应视为用户数据。

## 可重建索引

SQLite 历史索引用于搜索、历史浏览和同日年轮等体验。该索引是可重建缓存；当索引损坏或缺失时，可从 Markdown 条目、raw-input jsonl 文件和版本快照重新构建。

## 默认位置

默认情况下，Journal 的用户数据位于：

```text
%LocalAppData%/Journal
```

安装、升级和卸载流程应将该目录视为用户数据目录，而不是一次性安装产物。卸载应用时应默认保留该目录，避免误删个人日记。

## 导入与导出

导入数据包时，Journal 应先备份当前数据，再替换长期数据材料。数据材料恢复完成后，应重建 SQLite 索引，使历史搜索和同日年轮重新对齐本地文件。

导出数据包默认不包含完整 API Key。需要迁移 LLM 配置时，用户应重新确认 provider、密钥来源和可见性边界。

## 备份责任

Journal 尽量维护清晰的本地数据边界，但本地磁盘、系统账号、备份介质和第三方 LLM provider 不在应用完全控制范围内。重要日记应由用户自行保留可靠备份。
